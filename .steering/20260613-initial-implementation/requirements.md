# 初回実装 要求内容

## 1. 概要

`docs/` で定義したMVP仕様に基づき、書籍管理システムの初回実装を行う。
ユーザー登録・ログイン、書籍のCRUD、タグ管理、一覧・検索・並び替え・ページネーションまでをスコープとする。

## 2. 変更・追加する機能

### 2.1 認証・ユーザー管理
- ユーザー登録（email, password）
- ログイン（JWT発行）
- フロントエンドでの認証状態管理・未ログイン時のリダイレクト

### 2.2 書籍管理（CRUD）
- 書籍の登録・編集・削除・詳細表示
- データ項目: タイトル、著者、ステータス、評価、タグ、メモ

### 2.3 タグ管理
- タグの一覧・登録・編集（リネーム）・削除
- 書籍登録・編集時のタグ選択（サジェスト・新規作成）

### 2.4 一覧・検索・並び替え
- 書籍一覧の表形式表示・ページネーション
- タイトル・著者によるフリーテキスト検索
- タイトル・著者・登録日・評価による並び替え

## 3. ユーザーストーリー

`docs/product-requirements.md` の「6. ユーザーストーリー」を初回実装のスコープとする（全件対応）。

## 4. 受け入れ条件

`docs/product-requirements.md` の「7. 受け入れ条件」を初回実装の受け入れ条件とする。
加えて、以下を満たすこと。

- バックエンド: `ruff check`, `ruff format --check`, `pytest` が成功する
- フロントエンド: `npm run lint`, `npm run typecheck`, `npm run test` が成功する
- `docs/architecture.md`・`docs/repository-structure.md`・`docs/development-guidelines.md` で定義した構成・規約に従っている

## 5. 制約事項

- 外部の書誌情報APIとの連携は行わない（手入力のみ）
- タグのマスタ共有はユーザー単位（他ユーザーとは共有しない）
- DBはSQLiteを使用し、マイグレーションはAlembicで管理する
- 認証はJWT（Bearerトークン）方式とし、サーバーサイドセッションは持たない
